## Summary

Adds a small `Upcoming / In progress / Complete` badge with the season year + format label to the league detail header (layout-scoped, visible on every tab). Derived from the league's current season's phases filtered by `league.seasonFormat`, compared against `getAppNow()`.

## Business Spec Coverage

- §3.7 (new) — Season State Display, with the three-state matrix.

## Decisions

- **Badge lives on the layout header, not per-tab.** Every tab benefits from the same signal; duplicating per-page would drift.
- **Three distinct states** — `upcoming` (every relevant phase starts after now), `in_progress` (any relevant phase contains now), `complete` (every relevant phase has ended). The gap between regular-season week 18 and wild card still counts as `in_progress` for Full Season because wild card sits inside the span between earliest start and latest end.
- **Pure function in `lib/nfl/leagues.ts`.** Takes phases + format + now. Caller (the layout) fetches phases for the current season and passes everything in. No time injection in `lib/` — matches the existing pattern.
- **Fallback behavior.** If no current season exists (pre-setup DB, no phases synced), the badge hides entirely rather than rendering "?". Matches the invariant that leagues can't be created without a season.
- **No separate data query for "format-relevant phases".** Filtering happens in the lib function, keeps the data layer generic.

## Deviations from the Story

- None.

## Notable / Gotchas

- **Pro Bowl filtering is already upstream** in `lib/espn/nfl/phases.ts` — the phases list this helper gets never contains a Pro Bowl row, so `seasonFormatToSeasonTypes` doesn't need a special case.
- **Regular Season league, postseason in progress → Complete.** Intentional — a Regular Season league doesn't follow playoffs. Covered by a dedicated test.
- **Layout does three extra queries** (seasons, selectCurrentSeason is pure, phases-by-season). One DB round-trip for seasons, one for phases; acceptable since the layout already fetches league + member. Could collapse to a single joined query if the pages get chatty.
