## Summary

Adds the app's visual brand: an inline SVG logo mark + wordmark, programmatic
favicon / apple-icon / OG image via `ImageResponse`, and root-layout metadata
for Open Graph and Twitter. Headers across `(app)`, `(public)`, and `(legal)`
now render the logo instead of plain text. Login card gets the mark above the
title.

## Business Spec Coverage

N/A — pure branding. Driven by the UI rules' "sports fan feel — bold, energetic"
direction.

## Decisions

- **Single theme-adaptive SVG, not two light/dark files.** The story says
  "Logo (SVG) — light + dark variants". `components/brand/logo-mark.tsx` uses
  `fill-primary` / `fill-primary-foreground`, so it tracks the existing CSS
  tokens automatically. One source of truth; no drift between variants when
  the palette changes.
- **Programmatic icons over static files.** `app/icon.tsx`, `app/apple-icon.tsx`,
  and `app/opengraph-image.tsx` use `next/og` `ImageResponse`. Next 16 wires the
  `<head>` tags automatically via file conventions, so the root layout does not
  set `icons:` in its metadata.
- **Deleted `app/favicon.ico`** (the default Next starter icon). `app/icon.tsx`
  supersedes it; keeping both would give Next two competing root icons.
- **Hard-coded brand hex in `ImageResponse` files** (`#EA5323` for primary,
  plus OG card neutrals). Satori can't read CSS custom properties, so there's
  no way to reference `--primary` from these files. `#EA5323` is an sRGB
  approximation of the theme's `oklch(0.62 0.22 34)`. If the brand orange ever
  changes, update all three files.
- **Single shared description constant** in `app/layout.tsx` for `description`,
  `openGraph.description`, and `twitter.description` — avoids drift.
- **`metadataBase` from `NEXT_PUBLIC_SITE_URL`** with `http://localhost:3000`
  fallback. The env var was already in `.env.example`; the fallback keeps dev
  working without manual setup.
- **Mark design: bold "P" on a rounded-square badge.** Readable at 32×32
  favicon scale, keeps sports-fan feel. The "P" glyph is a hand-drawn SVG path,
  not a rendered font glyph, so it stays crisp at any size and doesn't depend
  on the end user's installed fonts.
- **`LogoMark` exposes an optional `title` prop** to control a11y: decorative
  usage (inside a labeled link) passes nothing and the SVG gets `aria-hidden`;
  standalone usage (login card) passes a title so screen readers announce it.
- **Added `.claude/scheduled_tasks.lock` to `.gitignore`.** Runtime artifact
  (session id, pid, timestamp) — showed up in `git status` during this task
  and was worth ignoring alongside `.claude/settings.json`.

## Deviations from the Story

- Story lists "Logo (SVG) — light + dark variants" as two artifacts. Delivered
  as one theme-aware SVG component instead. See decisions.
- Story says "app/icon.tsx or app/favicon.ico — favicon". Went with `icon.tsx`
  exclusively and removed the starter `favicon.ico`.
- Story does not mention updating the header chrome. Did it anyway — a logo
  that isn't in the header is half-shipped. Touches `AppShell`, `(public)`
  layout, `(legal)` layout, and the login page.

## Notable / Gotchas

- **`ImageResponse` uses Satori, not the full DOM.** Only a subset of CSS works
  (flexbox yes, grid no; most colors yes, CSS vars no). If you add more styles
  to the OG card, verify locally by hitting `/opengraph-image` in dev — the
  build won't catch Satori-unsupported properties.
- **Icon URLs are cache-busted by Next** with a content hash suffix
  (`/icon?97556bcfceac6300`). The browser cache key changes whenever the
  generated image changes. No action needed; just don't be surprised by the
  query string in `<link rel="icon" href=...>`.
- **Verified routes in dev:** `/icon` → 200 `image/png` 527B, `/apple-icon` →
  200 `image/png` 2.1KB, `/opengraph-image` → 200 `image/png` 38KB; `<head>`
  includes the expected `<link rel="icon">`, `<link rel="apple-touch-icon">`,
  and full `og:image` / `twitter:image` tags.
- **Brand hexes are duplicated across three `app/*.tsx` files.** Considered
  extracting to `lib/brand/colors.ts` but three call sites is under the
  abstraction bar; revisit if a fourth lands.
