## Summary

Turns the members page from a stub into the real member list and adds promote / demote / remove actions. Commissioners see a dropdown per row with role changes and (when the league is not in-season) a remove option. The sole-commissioner self-demote guard is enforced server-side; the "remove yourself" path routes the user to the Leave League flow (PL-026).

## Business Spec Coverage

- §4.1 — Role enum (commissioner / member) already in place since PL-020.
- §4.2 — Promote any member; demote any commissioner; self-demote blocked when sole commissioner.
- §4.3 — Remove other members only when not in-season; historical standings untouched (FK on `user.id`, not `leagueMembers.id`).
- §12.4 — Members tab list with roles and commissioner management options.
- §13.1 — Commissioner-only role changes + removals; in-season guard on removal.

## Decisions

- **Remove action rejects self-removal with a pointer to Leave League.** Cleaner separation of concerns — `removeMemberAction` is "commissioner removes someone else"; `leaveLeague` (PL-026) is "member removes themselves with additional cleanup rules". Self-removal path returns a business error pointing at the other flow.
- **Sole-commissioner guard only fires on self-demote.** Demoting a different commissioner always implies ≥2 commissioners existed, so the check is skipped on non-self demotes (cheaper, and the failure state is impossible).
- **Promoting an already-commissioner returns an error, not a silent success.** Keeps the UX honest — if a commissioner double-clicks and the UI refreshes, they still see a clear "already a commissioner" toast rather than a misleading "success".
- **Members list is visible to all members, not just commissioners.** §12.4 says the tab shows roles to everyone; the dropdown is commissioner-only.
- **`canRemove` is derived server-side** and passed to the Client Component. Keeps the business rule (`!inSeason`) out of the component.
- **Revalidate `/home` and `/leagues` on remove** so the removed user's layouts refresh on their next view. Commissioner's current tab refreshes via the standard league layout revalidation.

## Deviations from the Story

- None.

## Notable / Gotchas

- **`removeMemberAction` auth order** — validate → auth → authorize → self-check. The self-check lives AFTER `assertLeagueCommissioner` so non-commissioners always get `ForbiddenError` instead of the "Use 'Leave league'…" business error. Matches the documented pipeline.
- **`MembersList` is a Client Component only for the dropdown interactions.** The avatar + badge + name rendering are all composed here because colocation reads better than splitting it, but it still receives all its data as props from the server page.
- **Sole-commissioner check reads commissioner count lazily.** One extra round-trip only when self-demoting; skipped entirely when demoting someone else.
- **No atomicity around role updates.** A race where a commissioner demotes themselves while another commissioner leaves could leave the league with zero commissioners. Unlikely at MVP scale; consider a transactional `SELECT FOR UPDATE` when concurrency ramps up.
- **`updateLeagueMemberRole` on an already-targeted role** is effectively a no-op at the DB. The "already commissioner" pre-check saves the write and surfaces a cleaner message, but an alternate implementation could let the write happen and return success.

## Follow-ups
