## Summary

Live scores sync function that fetches current scores and game statuses from ESPN, updates event records, and tracks how many events finalized. Includes NFL season month gating and game window detection to avoid unnecessary API calls.

## Business Spec Coverage

BUSINESS_SPEC.md ┬¦10 (Live Scores & Game Results), ┬¦10.2 (Final Scores). BACKGROUND_JOBS.md ┬¦4 (Live Scores Job), ┬¦2 (Self-Gating).

## Decisions

- **No separate outcomes table**: BUSINESS_SPEC originally described an "outcome record" for confirmed final scores. Decided to use the event table's existing `status`/`homeScore`/`awayScore` fields directly ŌĆö when `status = 'final'`, the event's scores are authoritative. Simpler schema, one fewer table, same semantics. Updated BUSINESS_SPEC ┬¦10.2 accordingly.
- **`isGameWindowActive` is a pure function**: Takes an array of events and a `now` parameter rather than querying the DB itself. The sync function queries events then passes them in. This keeps the function testable without mocking and keeps data access in the sync layer.
- **`getScorableEvents` filters non-final events with valid refs**: Uses an inner join between `events` and `external_events` with `status != 'final'` and `IS NOT NULL` on all three ref columns. This ensures we only attempt ESPN fetches for events we can actually score.
- **`upsertDataSource` for ESPN lookup**: Matches the pattern in `setup.ts` ŌĆö idempotent and avoids adding a separate `getDataSourceByName` function.

## Deviations from the Story

- **"Creates outcome records when games go final"**: Replaced with updating event status to `final` directly. No outcomes table.
- **"Calls standings recalculation"**: PL-015 is a separate story. The sync function logs the count of finalized events so PL-015 can plug in later.

## Notable / Gotchas

- **`EventStatus` type was duplicated**: `lib/espn/nfl/scores.ts` (from PL-010) redeclared `EventStatus` instead of importing from `lib/db/schema/sports`. Fixed in this story.
- **Type assertion in `getScorableEvents`**: The `WHERE IS NOT NULL` clause guarantees non-null ref columns, but Drizzle's TypeScript types still report `string | null`. A `as ScorableEvent[]` cast is used with a comment explaining why it's safe.
- **`isNflSeasonMonth` uses JS month index**: `getMonth()` returns 0-based months. The Set `[0, 1, 7, 8, 9, 10, 11]` covers Jan, Feb, AugŌĆōDec.
