## Summary

Adds `updateLeagueAction` + the `/leagues/[leagueId]/settings` page with a structural-field lock whenever the league is in-season. Adds `getActivePhasesForSportsLeague` to the data layer (time window lives in SQL), simplifies `isLeagueInSeason` to a pure set-membership check on already-filtered phases, and reuses the create-league validators so form and action share the same rules.

## Business Spec Coverage

- §3.2 — Name/image editable any time; `seasonFormat`, `size`, `picksPerPhase`, `pickType` locked when in-season; `size` can't go below the current member count.
- §3.3 — In-season = any phase of a type in the league's format currently contains `now`.
- §13.1 — Commissioner gate on every edit; structural fields additionally gated on "Not in-season".

## Decisions

- **`isLeagueInSeason` no longer takes `now`.** The data layer (`getActivePhasesForSportsLeague(sportsLeagueId, now)`) is now the single place that compares against the clock; `lib/` only checks set membership. This cleans up `lib/` tests, removes a test-time injection point that was easy to get wrong, and keeps the semantic "is this active?" question where the DB's `now` lives.
- **Size check runs only when `size` changes.** Rereading BUSINESS_SPEC §3.2, the constraint is on lowering — unchanged size is a no-op. Tests cover both branches.
- **Name-only edits are always allowed.** `structuralChanged` is a boolean diff — only when a structural field differs from the stored value do we check `inSeason`. Covered by a test using an in-season active phase.
- **Client also disables structural inputs when `inSeason`.** Server still enforces, but the form hides the footgun instead of letting users bounce off a business error. `disabled` on the `<Select>` + `<Input type="number">` is a UX-only affordance — the action is authoritative.
- **`Promise.all` for phases + member count.** Both fetches are independent; fire in parallel. `getLeagueById` stays sequential because the subsequent queries need `league.sportsLeagueId`.
- **Schema reuse.** `updateLeagueSchema` composes the same `nameSchema` / `sizeSchema` / etc. as `createLeagueSchema` plus a `leagueId: z.string().uuid()`. One spot for bounds.
- **Settings page uses `PageProps<"/leagues/[leagueId]/settings">`.** Specific route type after `pnpm exec next typegen`.

## Deviations from the Story

- Backlog called the in-season helper `lib/nfl/scheduling.ts#isLeagueInSeason`. Landed as `lib/nfl/leagues.ts#isLeagueInSeason` to keep it adjacent to `seasonFormatToSeasonTypes` / `selectCurrentSeason` (all set up in PL-020). `scheduling.ts` stays the phase-math module.

## Notable / Gotchas

- **Zod v4 strict UUIDs.** `z.string().uuid()` rejects all-zero UUIDs because the version nibble has to be 1–8. Tests need a real v4-shaped string (`11111111-1111-4111-8111-...`). Same gotcha will hit PL-023/PL-024 fixtures.
- **Member-count race.** The size check reads `getLeagueMemberCount` and then writes via `updateLeague` without a transaction. A concurrent invite acceptance could slip past, pushing membership over the new cap. Acceptable today; worth wrapping in `withTransaction` once invite flows ship (PL-023/PL-024).
- **Pro Bowl exclusion is upstream.** `isLeagueInSeason` doesn't need special-casing because `lib/sync/nfl/setup.ts` + `lib/espn/nfl/phases.ts` filter Pro Bowl out before rows ever hit `phases`. If a future admin override accidentally inserted a Pro Bowl phase, in-season detection would fire for it.
- **`updateLeague` still throws `NotFoundError`** if the league was deleted between the read and the write. Currently uncaught in the action; caller would hit the error boundary. Left as-is — the window is tiny and correct behavior is "error", not "silent no-op".

## Code-reviewer findings (non-blockers not fixed)

- **`Math.max(2, memberCount)` duplicated** in the size `<Input min>` and description text. Stylistic; extract a local const if a third use lands.
- **`getLeagueById` is called by both `layout.tsx` and `settings/page.tsx`** for the same request. Next.js memoizes native `fetch`, not arbitrary function calls. Low cost; worth a cache helper once more pages need the same lookup.
- **Transactional size check** deferred (see Notable above).

## Spec-tracer findings

- No divergences. One informational note: the phase window semantics are `[startDate, endDate)` (inclusive start, exclusive end). BUSINESS_SPEC §3.3 says "falls within" without picking a side; our choice matches prior behavior.
