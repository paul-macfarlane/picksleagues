## Summary

Adds `leaveLeagueAction` plus a "Leave league" section on the members tab. Handles the three §4.4 branches server-side — sole-member leave deletes the league (cascading members/standings/invites), sole-commissioner leave is blocked unless they're also the sole member, everyone else can leave normally. All branches short-circuit on in-season state.

## Business Spec Coverage

- §4.4 — Not-in-season AND (sole member → delete league, else not-sole-commissioner).
- §13.1 — Any member can leave under the same conditions.

## Decisions

- **Business rule order: in-season → sole-member → sole-commissioner → leave.** Matches the spec wording exactly. In-season block runs first so it even blocks sole-member delete.
- **`removeLeague` uses FK cascade.** Members, standings, direct invites, and link invites all cascade — no manual cleanup in the action.
- **Sole-member delete uses the same primitive as `deleteLeagueAction`.** `removeLeague` throws `NotFoundError` if the row is gone; we don't catch here because `assertLeagueMember` would already have returned a `ForbiddenError` in that case.
- **Leave button variant flips to `destructive` when sole-member.** Signals the true weight of the action ("Leaving deletes the league.").
- **UI also hides the Leave section when in-season.** Defense-in-depth — the server still enforces. Matches the pattern used for Invites and Members removal on the same page.
- **No transaction wrapping.** The sole-commissioner guard (`getCommissionerCount`) reads before `removeLeagueMember`; a concurrent promote/demote could theoretically race, but the cost of serializing is high and the MVP contention is near-zero. Note in follow-ups for later hardening.

## Deviations from the Story

- None.

## Notable / Gotchas

- **Commissioner race.** Commissioner A is demoting themselves, Commissioner B is leaving — both read `commissionerCount = 2`, both proceed, league ends up with zero commissioners. Same race as PL-025; documented there.
- **`isSoleMember` prop is cosmetic-only.** The server re-computes from `getLeagueMemberCount`. If the client and server disagree (member joined between render and click), the toast + destination reflect the server result via `leagueDeleted`.
- **Sole-member-but-in-season is blocked.** Spec §4.4 is unambiguous: `not in-season AND (sole-member OR not-sole-commissioner)`. Sole members in-season need to wait for the season to end — or use `deleteLeagueAction` (commissioner-only, no in-season gate). Surfaces correctly as "can't leave while in-season".
- **`assertLeagueMember` returns the member record**, which we reuse for the role branch at line 187. Saves a second `getLeagueMember` round-trip.
