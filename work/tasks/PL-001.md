## Decisions

- **Skipped `@neondatabase/serverless`.** The story listed it, but `BACKGROUND_JOBS.md` mandates Node.js runtime (pg driver). `pg` works fine against Neon from Node.js serverless functions. If we hit connection-pool issues at scale we can add the Neon HTTP driver conditionally in PL-060/061; adding it preemptively would double the driver surface for no near-term benefit.
- **shadcn preset: `-t next -b radix -p nova`.** Radix primitives per `TECH_STACK.md`; Nova is the current default aesthetic preset.
- **Kept the monolithic `radix-ui` dep.** The radix-nova preset imports from it directly (`import { Dialog as DialogPrimitive } from "radix-ui"`) rather than the per-primitive `@radix-ui/react-*` packages. Removing it would break every generated component.
- **`vitest.config.ts` → `passWithNoTests: true`.** Avoids a red CI signal until the first test file lands in a later story.
- **Base color deferred to PL-005.** `shadcn init` used the default `neutral`; PL-005 owns the sports-fan aesthetic and will override.

## Notable / Gotchas

- **`shadcn/form` is a deprecated stub in the radix-nova preset.** `shadcn view @shadcn/form` returns an empty registry entry (no files, no deps). The modern replacement is the **Field** component family — `<Field>`, `<FieldLabel>`, `<FieldDescription>`, `<FieldError>`. RHF integration uses `errors={[fieldState.error]}` on `<FieldError>`. `rules/ui.md` and `docs/ARCHITECTURE.md §5` were updated to reflect this; if you see `<FormMessage>` / `<FormField>` in external examples, translate before copying.
- **`shadcn add` CLI is quiet on no-op.** Adding an already-deprecated component (like `form`) shows "Checking registry... ✔" and exits with no files created and no error. Verify with `ls components/ui/` after each add.
- **`shadcn init` overwrites `app/globals.css`.** PL-005 will rewrite it for the sports-fan theme; no data loss, but don't hand-edit globals.css until PL-005 has done its pass.
