## Summary

Added `app/(app)/admin/simulator/page.tsx` with status display, Initialize / Advance / Reset controls, and a silent-redirect admin guard via `app/(app)/admin/layout.tsx`. Server Actions live in `actions/simulator.ts` and follow the standard pipeline (validate → auth → authorize → execute → revalidate → return). Destructive reset uses the AlertDialog confirmation pattern already established by delete-account-dialog.

## Decisions

- **Admin gate in `(app)/admin/layout.tsx` via `redirect("/home")`** — silent redirect keeps the admin panel hidden from non-admins. Server Actions separately call `assertCallerIsAdmin()` → `assertAdmin(profile)` (throws `ForbiddenError`) for defense-in-depth; layout guards rendering, actions guard mutations.
- **Form uses react-hook-form + zodResolver** — per the forms rule in `rules/ui.md`. `initializeSimulatorSchema` is shared between the form and the action.
- **Dropped `z.coerce.number()` in favor of `z.number().int()`** — RHF's `valueAsNumber: true` already converts input strings to numbers, and coerce produced an input/output type mismatch with `useForm<InitializeSimulatorInput>`. The form is the only caller; server-side direct invocations are not a use case.
- **Single source of truth for year-range enforcement** — the check lives in `lib/simulator.ts:isValidSimulatorYear` and bubbles as `BadRequestError`. Actions don't re-implement the rule. (Fix from the code-reviewer on the first pass.)
- **Reset dialog mirrors `components/account/delete-account-dialog.tsx`** — same AlertDialog shape, `event.preventDefault()` on the destructive action, `useTransition` + sonner. Consistent destructive-confirm UX across the app.
- **Status card is a Server Component** — no interactivity; simply formats props from `getStatus()`. Date formatting uses `formatInTimeZone` from `date-fns-tz` in line with the rest of the codebase.
- **Standings state display deferred** — called out in the "Reset" card copy ("Standings display is planned for a later story") so it's not forgotten.

## Notable / Gotchas

- **`z.coerce.number()` breaks RHF generics**. With coerce, zod's input type is `unknown` and output type is `number`, which confuses `useForm<T>` (expects a single type). Drop coerce at the schema level and let `valueAsNumber: true` in `register` handle client-side conversion.
- **Admin panel layout re-fetches profile**. The outer `(app)/layout.tsx` already loads the profile, but Server Component layouts can't pass data down to children. A separate fetch in `admin/layout.tsx` is the cleanest option; profile queries are cheap.
- **Action's `assertCallerIsAdmin` helper is local**. Will want to hoist into `lib/permissions.ts` (as `requireAdminSession()`) when a second admin action lands — today it's the only caller, so it stays local.
- **No tests for the page / layout / components** — per `rules/testing.md`, we don't unit-test React components or Next routing. Action coverage (auth, authz, validation, business-error mapping, success, unexpected propagation) is what the test suite verifies.
