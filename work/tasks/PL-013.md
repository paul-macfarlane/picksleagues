## Summary

Added the recurring odds sync pipeline that fetches current spreads/moneylines/over-under from ESPN for unstarted events during active game windows.

## Business Spec Coverage

BUSINESS_SPEC §9 (Odds & Spreads), BACKGROUND_JOBS §3 (Odds Sync Job).

## Decisions

- **Query-only for reference data**: Sync jobs use `getDataSourceByName` / `getSportsbookByName` instead of upserting. Setup owns creation of reference data; sync jobs only read it. Also applied this retroactively to live-scores sync (PL-012).
- **No `upsertExternalSportsbook` in sync loop**: The external sportsbook bridge record is created during setup. Odds-sync uses `sportsbook.id` directly, avoiding unnecessary DB round-trips.
- **Game window check uses synthetic status**: `getOddsSyncableEvents` only returns `not_started` events, so we construct `{ status: "not_started" }` objects to satisfy `isGameWindowActive`'s type signature.

## Notable / Gotchas

- `getDataSourceByName` and `getSportsbookByName` were added to `data/sports.ts` as new query functions. They throw `NotFoundError` if the record doesn't exist — this is intentional, as setup must have run first.
- The live-scores sync was also updated in this commit to use `getDataSourceByName` instead of `upsertDataSource` for consistency.
