## Summary

Adds the `link_invites` table, the `createLinkInvite` / `revokeLinkInvite` / `joinViaLinkInvite` actions, and the `(public)/join/[token]` page. Extends `cleanupInvitesIfFull` so both direct and link invites are wiped when a join fills the league. Login page now accepts a `?redirect=` search param for "sign in → return to invite link" flows.

## Business Spec Coverage

- §5.2 — Commissioner creates a shareable link (random UUID token); anyone can use it; preview w/o auth; join requires auth; role + 1–30d expiration default 7.
- §5.4 — Capacity-filling join now wipes direct AND link invites.
- §5.5 — Link invites have a revoke button; direct-invite revoke UI is still deferred to PL-025.
- §12.6 — `/join/[token]` shows "invite expired", "sign in to join", the league preview + Join button, and "already a member / go to league" states.
- §13.1 — Commissioner-only create; any authenticated user can join via valid link.
- §8.6 — Join inserts the zeroed standing for the current season in the same transaction as the member insert.

## Decisions

- **Token = `randomUUID()`.** Fine for MVP: 122 bits of entropy is unguessable in practice, and UUIDs are already in the dependency graph. Drop-in swap to nanoid later if URLs need to be shorter.
- **Link invites are NOT consumed on join.** Per §5.2 "anyone with the link can use it (no per-user limit)" — we only delete invites via revoke, expiry, or the capacity-fill cleanup.
- **Already-a-member short-circuit.** If a user clicks the link and is already a league member, the action returns `{ success: true, data: { leagueId } }` instead of an error — the UI just routes them to the league page. Silent happy-path; avoids confusing "you're already in" errors.
- **`?redirect=` sanitisation.** Allow only same-origin paths (`/^\/[^/\\]/`) so `//evil.com` / `/\evil.com` both fall back to `/home`. Regex is stricter than PL-023 "starts with `/`" to block the protocol-relative attack.
- **Preview page uses `auth.api.getSession` directly**, not `getSession()`. The preview has to tolerate unauthenticated visitors without throwing `UnauthorizedError`.
- **Cleanup wipes BOTH tables via `Promise.all`.** One helper, symmetric. Exercised by tests on both the direct-invite accept path and the link-invite join path.
- **No `withTransaction` around the link-invite insert.** Only one write (the invite row); no atomicity concern. The join path stays transactional because member + standing must land together.
- **Commissioner "Copy link" copies the full `${origin}/join/${token}` URL**, generated client-side at click time so it works regardless of the server URL configuration.

## Deviations from the Story

- Backlog said "Auto-cleanup of invites when league reaches capacity" — extended PL-023's helper rather than duplicating the logic. Both invite tables are now symmetric.

## Notable / Gotchas

- **`react-hooks/purity` forbids `Date.now()` in component render.** `new Date()` comparison works because the rule doesn't see through constructor calls. Expired check at render time is thus `invite.expiresAt <= new Date()` — functionally identical, lint-passing.
- **`LinkInvitesSection` list shows expired invites** (with "Expired" copy and a disabled Copy button, Revoke still enabled). Commissioners need a way to clean up expired rows. Matches §5.5 intent; deferred from full "active invites" UX.
- **No rate limit on `createLinkInviteAction`.** A commissioner could spam links. Low-severity; combine with a rate-limit rollout for other mutations later.
- **`getLinkInvitesByLeague` doesn't filter by expiration**, intentionally — the commissioner view wants to see recent history including expired rows. Revoke still works on expired rows.
- **`/join/[token]` is a Server Component that awaits `auth.api.getSession`.** If the BetterAuth session endpoint is flaky, the page will error. Acceptable for MVP; the session read is cached per-request in BetterAuth.
- **Already-a-member check happens AFTER `getSession`.** The `joinViaLinkInviteAction` can still throw `UnauthorizedError` before hitting that branch — expected.
