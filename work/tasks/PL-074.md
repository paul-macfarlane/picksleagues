## Summary

Added `updateOddsAction` and an edit dialog on the Odds tab of `/admin/overrides`. Admin can set spreads / moneylines / over-under per existing `(event, sportsbook)` row; empty fields save as `null`. Save auto-sets `lockedAt`.

## Business Spec Coverage

N/A — admin ops tooling.

## Decisions

- **Edits only touch existing rows.** Per the epic planning Q3, no flow for adding odds on a new sportsbook. Today only ESPN Bet is seeded, so the odds table always has one row per event (or zero).
- **Empty fields → null.** Matches the nullable `doublePrecision` / `integer` columns. Admins can clear a broken value without having to enter zero.
- **Spreads and over/under use `decimalString`; moneylines use `integerString`.** Regex-based validation keeps the schema shape identical to the other admin validators (no transform). `parseDecimal` / `parseInteger` convert after `safeParse`.
- **No business-rule coupling between home/away spreads.** Classic sportsbooks show `home = -away`, but admin corrections may legitimately diverge. Left it to the admin.
- **Placeholder on moneyline Inputs says `"e.g. 150"`** (no leading `+`). The integer regex rejects `+150`; using a `+` in the hint would lead the admin into a validation error.

## Deviations from the Story

- None.

## Notable / Gotchas

- **Mid-typing decimals (`"3."`, `".5"`) fail the decimal regex.** The form validates on blur, so intermittent errors during typing aren't surfaced — but if an admin leaves a trailing dot or leading dot, they'll see "Enter a number." Acceptable edge case for an admin tool.
- **`overUnder` is a single float, not a paired line.** The DB column is one value; both "over" and "under" use the same number (e.g. 47.5 means over 47.5 or under 47.5). Admin doesn't need two fields.
- **Moneyline range.** DB column is `integer`; extreme values (e.g. `-9999999`) are technically accepted. In practice sportsbooks cap around ±5000, but the schema doesn't enforce a range.

## Follow-ups

- **Accept `+` in moneyline input** — strip a leading `+` before regex validation if we want to mirror sportsbook convention. Small polish.
