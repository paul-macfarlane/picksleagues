## Summary

Database schema for all NFL sports entities (seasons, phases, teams, events, odds) with external ID bridge tables, data layer functions, pick lock time calculation, and the sync orchestration module that populates the database from ESPN's API.

## Business Spec Coverage

BUSINESS_SPEC.md §6.2 (pick lock time), §11 (NFL data sync). BACKGROUND_JOBS.md §3 (initial setup job), §6 (external ID mapping).

## Decisions

- **Sport-agnostic core tables with `sportsLeagueId` FK**: `seasons`, `phases`, `teams`, `events`, `odds` are not NFL-prefixed. Each links to a `sportsLeagues` row. Future sports add rows to `sportsLeagues`, not new table sets. Bridge tables remain provider+sport specific (`external_*`).
- **No `external_odds` bridge table**: `rules/database.md` lists it but odds don't have standalone ESPN IDs — they're fetched via `$ref` URLs stored on `external_events`. The event bridge stores `oddsRef` for this purpose.
- **Phase dates snap to Tuesday 2 AM ET boundaries**: Applied during phase sync (from ESPN's raw week dates) and refined after event sync (from actual game times). Uses `fromZonedTime` with freshly constructed Date components — mutating `toZonedTime` results with `setDate`/`setHours` was unreliable across system timezones.
- **Only current season synced**: `fetchCurrentSeason` returns one season rather than syncing all ~100 historical seasons from ESPN. Re-running adds future seasons when ESPN publishes them.
- **Layer exception for `lib/sync/`**: Sync pipelines inherently orchestrate both external API calls (`lib/espn/`) and database persistence (`data/`). Added documented exception in `rules/architecture.md` alongside the existing `lib/permissions.ts` exception.
- **Concurrent ESPN fetches, sequential DB writes**: ESPN API calls for events and odds are parallelized via `Promise.all`. DB writes remain sequential — acceptable for an infrequent setup job and avoids connection pool pressure.

## Deviations from the Story

- **Added `sportsLeagues` table**: Not in the original story but needed for sport-agnostic schema per BUSINESS_SPEC §15 future features guidance.
- **Added `scripts/run-setup.ts`**: One-off script for manual testing since PL-014 (cron route handlers) is a later story. Uses dynamic import to load env vars before DB client init.

## Notable / Gotchas

- **ESPN 2026 events not published yet**: As of April 2026, ESPN returns empty event lists for the 2026 season. Phases sync with correct Tuesday boundaries from ESPN's week metadata. Events/odds will populate when ESPN publishes the schedule (typically May/June).
- **`toZonedTime` + `setHours` is a footgun**: The original implementation mutated `toZonedTime` results, which produced wrong times depending on the system timezone. Fixed by extracting Eastern calendar components with `getEasternComponents()` and constructing target times directly with `fromZonedTime(new Date(y, m, d, h), tz)`.
- **`$onUpdate` on `updatedAt` columns**: Drizzle's `$onUpdate` runs in the ORM layer, not the DB. The data layer also passes `updatedAt: new Date()` explicitly in `set` clauses for upserts/updates to ensure it's always current.
