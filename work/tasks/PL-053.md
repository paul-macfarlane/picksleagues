## Summary

Adds a public splash page at `/` (hero, three feature cards, secondary CTA)
that describes the product to unauthenticated visitors and routes them into
the OAuth flow. Authenticated visitors are redirected past the splash to the
authed home, which now lives at `/home`.

## Business Spec Coverage

- §1 Product Overview — the splash's hero copy and feature cards pull
  directly from the NFL Pick'Em value prop (private leagues, weekly
  straight-up / ATS picks, season-long leaderboard).
- §12.1 Onboarding — splash is the step immediately upstream of the login
  page in the onboarding funnel.

## Decisions

- **Moved the authed home to `/home`, kept `/` as the public splash.**
  `app/(public)/page.tsx` and `app/(app)/page.tsx` both resolve to `/` —
  Next.js collapses route groups into the same URL namespace. The story
  explicitly specifies `app/(public)/page.tsx`, so the splash wins `/` and
  the old `(app)/page.tsx` welcome placeholder was moved to
  `(app)/home/page.tsx`. Alternative (conditional root route that checks
  session inline) was rejected because it violates the `(public)`/`(app)`
  route-group split the backlog and rules are built around.
- **Redirect chain lives in the page, not middleware.** `(public)/page.tsx`
  calls `getSession()` / `getProfileByUserId` and `redirect`s authed users
  to `/home` (or `/setup` if their profile isn't complete). Mirrors the
  pattern already in `(app)/layout.tsx` and `(setup)/layout.tsx`. Keeps the
  auth surface contained — no new middleware file — and lets the public
  layout render for actually-public visitors with zero extra round trips.
- **Extracted `SplashContent` as a local helper.** The redirect branch
  returns early, the render branch returns `<SplashContent />`. Keeps the
  `export default` function short and makes the control flow obvious. Not a
  new component — it lives in the same file and is never exported.
- **Three features, not more.** `Lock` / `ClipboardList` / `Trophy` from
  `lucide-react` (already a dep), each one card. Sports-fan tone without
  dumping every §7/§8 detail on the landing page.
- **Copy pulled verbatim from §1.** Hero subhead is a lightly-edited paste
  of the spec: "Create or join a private league, make weekly picks, climb
  the season-long leaderboard." Keeps the splash and the spec in sync — if
  the value prop ever changes, update §1 and copy it down.
- **Used shadcn `Card` + `CardHeader`-only for features.** No body/content
  section because each card is a single sentence. `CardHeader` carries icon
  + title + description in a compact stack.
- **Privacy/Terms via the existing `SiteFooter`.** The story lists "Links
  to login, privacy, and terms" — `(public)/layout.tsx` already renders
  `<SiteFooter />` which has the privacy + terms links. No new markup
  needed on the page itself.
- **Both CTAs point at `/login`.** Primary "Get started" (solid) in the
  hero, secondary "Sign in" (outline) in a closing section. Google/Discord
  brand buttons stay on the login page — duplicating them on the splash
  would add a third surface to maintain when we add a third provider.

## Deviations from the Story

- Story doesn't mention moving the authed home; this was the only way to
  land `(public)/page.tsx` at `/` without a route conflict. Touches
  `login-buttons.tsx` (`callbackURL`), `app-shell.tsx` (header logo link),
  and `(setup)/layout.tsx` (post-setup redirect) as a consequence.
- `(public)/layout.tsx` and `(legal)/layout.tsx` logo links intentionally
  **stay** `href="/"` — splash is the correct public home. Only the authed
  `AppShell` logo was repointed to `/home`.

## Notable / Gotchas

- **`/` gets the public layout now, not the app shell.** Anything
  expecting the authed header/nav at the root URL needs to use `/home`.
  The `callbackURL: "/home"` change in `login-buttons.tsx` is the
  load-bearing piece of the redirect chain; reverting it would drop
  fresh OAuth users onto the splash instead of their home.
- **Next typegen is stale after moving a `page.tsx`.** `pnpm typecheck`
  failed with `Cannot find module '../../../app/(app)/page.js'` until I
  deleted `.next/dev/types/` and reran `pnpm exec next typegen`. If you
  move or rename a route in the future and typecheck fails with a
  "validator.ts" error, nuke that directory.
- **`redirect()` inside a `try/catch` still works** as long as the catch
  rethrows non-`UnauthorizedError` exceptions — the redirect throws a
  `NEXT_REDIRECT` error which falls through the catch. The page uses the
  cleaner pattern (try only wraps `getSession`, redirects sit outside).
- **Verified routes in dev**: `/` → 200 with splash copy rendered,
  `/home` → 307 to `/login` when unauth (via `(app)/layout.tsx` guard),
  `/login` → 200. Authed flow not exercisable in pure dev without a real
  OAuth session.
- **Icons use `text-primary`.** Satori-rendered OG images still hardcode
  `#EA5323` (PL-009 carried a note about this); the splash renders in the
  real DOM so it can use the token.

## Follow-ups

- Splash will need real hero imagery or a stats strip once the product has

- If a third OAuth provider is added, decide whether the splash should
  carry provider-specific buttons or keep the "Sign in with Google or
