## Summary

Added `lib/cron-auth.ts` (bearer token verification + shared route wrapper), four POST cron route handlers under `app/api/cron/nfl/`, and `lib/sync/nfl/weekly-sync.ts`. Extracted the shared phase + team + event sync body out of `setup.ts` into `lib/sync/nfl/structural.ts` so setup (with odds) and weekly-sync (without odds) share one implementation.

## Business Spec Coverage

BACKGROUND_JOBS.md §Authentication, §Self-Gating, §Job Descriptions (Initial Setup / Weekly Sync / Odds Sync / Live Scores), §Job Schedule Summary.

## Decisions

- **`runStructuralSync` extracted into its own module** (`lib/sync/nfl/structural.ts`) and called by both `runInitialSetup` (wraps it with seed + odds) and `runWeeklySync` (wraps it with an `isNflSeasonMonth` guard only). One source of truth for phases + teams + events sync.
- **Weekly sync does not sync odds or live scores** — matches the spec description. Odds stay owned by the dedicated odds-sync job; live scores by the live-scores job.
- **`CRON_SECRET` must be set** — empty string or unset returns 500 ("CRON_SECRET is not configured"). No local bypass. Replaced the prior `.env.example` note that implied empty = no auth.
- **All cron routes are POST** with `export const maxDuration = 60`. Route names follow `ARCHITECTURE.md` + `work/Backlog.md` (`setup`, `weekly-sync`, `odds-sync`, `live-scores`). Updated `BACKGROUND_JOBS.md` to match (it previously used `sync-weekly` / `sync-odds` / `sync-scores`).
- **`handleCronRoute` wrapper** in `lib/cron-auth.ts` owns the auth check + try/catch + response shaping so each route file is a one-liner. Errors are logged to `console.error`; Sentry comes in PL-050.
- **New data-layer getter `getSportsLeagueByAbbreviation`** mirrors `getDataSourceByName` / `getSportsbookByName`. Weekly sync looks up reference records rather than upserting them (matches the `feedback_sync_no_upsert` project memory).
- **Deleted `scripts/run-setup.ts`** — superseded by `POST /api/cron/nfl/setup`. No `package.json` reference needed removal.
- **PL-015 is out of scope.** The live-scores sync still logs `"standings recalculation needed (PL-015)"` when events finalize; PL-015 will replace that log with the actual call when picks and standings schemas exist (Section 5). The re-slotted PL-015 entry now explicitly depends on PL-028, PL-030, PL-031.

## Notable / Gotchas

- **`vi.stubEnv` is the right way to manipulate `process.env` in tests.** The first draft used module-level snapshots + direct assignment; when `CRON_SECRET` is unset, `process.env.X = undefined` coerces to the string `"undefined"` and leaks into other tests. Fixed per the code-reviewer non-blocker.
- **`.env.example` is git-ignored** (`.env*` pattern in `.gitignore` catches it). The updated comment lives locally but won't appear in the PR. Pre-existing — tracked as a follow-up, not in scope here.
- **Pre-existing ambiguity in BACKGROUND_JOBS.md §Game Window Detection** — spec says "N hours", code uses `GAME_WINDOW_LEAD_MS = 30 minutes` (set in PL-012). Worth pinning to an actual number in the spec before the next sync-related story, but not a PL-014 change.
